Lessons Learned from Building My First Full-Stack Application

  @ Goutam Singla         September 25, 2024       Visit Site

Building my first full-stack application has been an incredible journey filled with challenges, breakthroughs, and valuable lessons. As a developer focused on Angular and Spring Boot, this experience has given me practical insights into both frontend and backend development. In this post, I’m sharing what I learned from creating my project, the Universal Shopping App, which integrates Angular for the frontend and Spring Boot for the backend.

Choosing the Right Tech Stack

When building my full-stack application, choosing the right tools was critical. I settled on Angular for the frontend and Spring Boot for the backend due to their robust ecosystems and scalability. For the database, I chose MySQL, which is a relational database management system that works well with Java-based backends like Spring Boot. Here's why:
  • Angular: Provides a structured framework, making it easier to manage complex applications. The standalone components approach kept my codebase modular.
  • Spring Boot: It allowed me to easily manage the backend, especially with its built-in support for RESTful APIs , database integrations and the integration of security features.
  • MySQL: This relational database was perfect for managing structured data such as users, products, and orders. Its compatibility with JPA (Java Persistence API) allowed me to define relationships and constraints between entities with ease.
Lesson learned: Your tech stack should not only suit your project’s current requirements but also scale with future enhancements.

Importance of Requirement Analysis

Before jumping into development, I had to clearly define what the application needed to do. Its the most important things that many beginners skip. The requirement analysis phase allowed me to:
  • Understand the needs of different user roles (customers and admins).
  • Define the core functionalities (product listing, cart management, order processing, and user authentication).
  • Create an outline of the user journey for both customers and admins.
This phase required creating use case diagrams, context-level data flow diagrams (DFD), and even an ER diagram to map out the data structures and how they would interact with each other. One key insight was how critical it was to fully understand the functional and non-functional requirements before starting development.
Lesson learned: Proper requirement analysis is the foundation for a successful project. It helped me avoid unnecessary rework and ensured that all key features were built with the end users in mind.

Setting Up Authentication and Authorization

In a multi-role application (customers and admins), implementing security is critical. I had to ensure that different users had access to specific resources:
  • JWT Authentication: I learned to handle token-based authentication in Spring Boot using a custom JwtRequestFilter for securing APIs.
  • Role-based Authorization: Managing roles like admin and customer became easier with well-structured backend logic.
However, I encountered issues while managing authentication between Angular and Spring Boot but eventually it is solved by breaking the problem in layers and debugging it layer by layer.
Lesson learned: Managing security, especially in full-stack projects, requires meticulous planning. Understanding how roles interact with API security is essential for building scalable, secure applications.

Frontend-Backend Communication: Handling API Requests

One of the most important aspects of any full-stack application is how the frontend communicates with the backend. I learned:
  • HTTPClient in Angular: This helped me make API calls to my Spring Boot backend. By using services in Angular, I was able to structure the code efficiently.
  • Error Handling: In a production app, handling errors gracefully is a must. I implemented common error responses like 401 Unauthorized or 500 Server Error, which helped in both debugging and providing feedback to users.
Lesson learned: Debugging communication between the frontend and backend requires an understanding of both sides. Logs in Spring Boot and the Angular console were extremely helpful.

Database Design and Data Relationships

For the backend, I needed to set up a relational database structure using JPA. Working with entities like User, Product, Order, Coupon and CartItems was a core part of the project. Some lessons included:
  • Handling different type of relationships between tables. eg: many-to-many relationship between orders and products.
  • data flow between the frontend, backend, and database
  • Database security using access controls and encrytion, implemented to protect sensitive user data
  • Implementing unique constraints for preventing duplicate records in the tables.
At one point, I faced issues with relationship between tables i.e. many-to-one or many-to-many , which was resolved by managing the tables structure more efficiently.
Lesson learned: Proper database design is crucial from the start. Having clear entity relationships reduces bugs and simplifies queries.

Frontend Development: Managing State and Components

Angular's standalone components made it easier to manage my frontend code by eliminating the need for a central app.module.ts. This structure helped me focus on each component’s functionality. Some of the lessons learned in frontend developement are:
  • Role-specific views: Admins and customers had different dashboards, which I handled using conditional rendering based on the user's role.
  • Services: : Implementing services that handles business logic, data retrieval, and other functionalities in a separate file and injecting it in components.
  • Routing: Handling Angular Routing enables navigation between different views or components in a single-page application (SPA). The Angular Router helps manage the URLs and the views associated with them.
I also dealt with managing local storage in Angular to store user tokens and session data. Encountering errors was part of the process, but careful debugging helped resolve these quickly.
Lesson learned: Angular's component-based architecture allows for a clean separation of concerns and also features like services , routing make it easier for developement.

Handling Deployment Challenges

Once the app was working locally, I had to deploy it. Deploying my full-stack application had its hurdles. I used GitHub for version control and deployed the frontend on Netlify and backend on Railway. I also faced some challenges:
  • Backend on Railway: Setting up the MySQL database connection was tricky at first, mainly due to misconfigured credentials in `application.properties`. Additionally, CORS issues arose because the frontend and backend were hosted on different domains. Configuring CORS in Spring Boot solved this.
  • Frontend on Netlify: The deployment was smooth overall, but I had to adjust the **API URL** in Angular’s production environment configuration to point to the Railway backend. I also added a `_redirects` file to fix client-side routing issues.
Lesson Learned: Deployment is not just about getting the code online. It requires managing different environments, making sure build processes work properly, routing adjustments and sometimes troubleshooting platform-specific issues.

Handling Reviews and Feedback

In developing the Universal Shopping app, I prioritized the importance of handling reviews and feedback from users. This process involved implementing a structured system for collecting, analyzing, and responding to user insights. By actively engaging with customer feedback, one can able to:
  • Enhance User Experience: Understanding users' perspectives helped identify pain points and areas for improvement, allowing me to make informed adjustments to the app's features and functionality.
  • Build Trust and Loyalty: Responding to reviews, whether positive or negative, fostered a sense of community and trust among users. Demonstrating that their opinions were valued encouraged continued engagement and loyalty.
  • Continuous Improvement: The feedback loop created a dynamic environment for ongoing development, ensuring that the app evolved in alignment with user needs and market trends.
Lesson learned: By integrating a robust feedback mechanism, I learned how crucial it is for any application to not only gather user reviews but also to act on them effectively, ultimately contributing to the app's success.

Conclusion

Building my first full-stack application was a significant milestone in my development journey. From managing the tech stack to deploying the application, each step taught me something valuable. It’s not just about writing code but understanding how every component interacts — from the database to the UI. As I move forward, I plan to improve my deployment pipelines, refine security measures, and explore more advanced features for future projects.
If you're embarking on your first full-stack project, my advice is to embrace the challenges and learn from every mistake, as each one brings you closer to mastering your craft.
